# MVC 
->  MVC 패턴 기반 MVT 
- 장고는 MVC(model, view, controller)를 기반으로 한 프레임워크<br/>
    - 컨트롤러(controller), 템플릿(template) => '뷰(views)' 라고 함 <br/>
        - 뷰(views)        :데이터를 가져오고 변형하는 컴포넌트 <br/>
        - 모델(model)      :데이터 베이스를 엑세스하는 컴포넌트 <br/>
        - 템플릿(template) :데이터를 사용자에게 보여주는 컴포넌트 <br/>
    - MVT 프레임워크라고 부르기도 함 => 용어만 다를뿐 개념은 동일 <br/>

+ 장고에서 뷰는 데이터를 가져오고 변형하는 컴포넌트<br/>
+ 템플릿은 데이터를 사용자에게 보여주는 컴포넌트<br/>

# 장고의 특징 

1. 구성요소들 간의 긴밀한 통합 <br/>
    - 장고는 각 요소가 긴밀하게 통합되어 있음  <br/>
    - 모든 부분이 통합적(integration)이며 동작이 빠르다. 그리고 재사용 가능하게 설계   <br/>

2. **객체관계의 매핑** <br/>
    - 객체 관계의 매핑(Object-Relational Mapper, ORM))이란 ? <br/>  
        : 데이터 베이스 엔진과 데이터 모델을 연결시키는 다리와 같은 역활   <br/> 
    - 직접적으로 SQL을 다루지 않아도 된다. 
    - 다양한 데이터 베이스를 지원, 유연한 데이터 베이스 시스템 변경     <br/>
    - 장고는 데이터베이스를 일관성 있게 추상화해서 단일한 API로 여러 데이터베이스를 사용 <br/>     

3. 간단한 URL 주소 설계(우아한 URL 설계)   <br/>
    - 강력하고 유연한 URL 주소 디자인  <br/>
         - URL주소 형태를 직접 결정 <br/>
         - 직관적이고 쉽게 표현할 수 있음 <br/>
         - 정규표현식을 사용하여 복잡한 URL도 표현가능 <br/>

    - 파이썬 함수를 각 주소 형태와 직접 연결 => (1:1) <br/>
        - 개발이 편리하며 이해하기도 쉬움<br/>
        - 일반 사용자와 검생 엔진 각각에 알맞는 결과를 제공 <br/>

4. 자동으로 구성되는 관리자 화면 <br/>
    - 데이터베이스에 대한 관리기능을 위하여 프로젝트를 시작하는 시점에 기본기능으로 관리자 화면이 제공 <br/>
    - 장고는 실행하는 순간 미리 내장되어 있는 관리자 화면 제공 <br/>
    - 여러 어플리케이션의 데이터를 쉽게 관리, 재구성 <br/>

5. 풍부한 개발 환경<br/>
    - 테스트용 서버를 포함 => 아파치 등의 웹서버가 없어도 테스트 진행 가능 <br/>
    - 가벼운 웹 서버를 포함하고 있고 개발뿐아니라 테스트 용도로 사용가능 <br/>
    - 디버깅 모드를 사용할 경우 장고는 에러를 쉽게 파악하고 해결할 수 있게 상세한 에러 메세지를 보여줌 <br/>

6. 자체 템플릿 시스템 <br/>
    - 내부적으로 확장이 가능하고 디자인이 쉬운 강력한 템플릿 시스템을 가지고 있음 <br/>
        - 화면 디자인과 로직에 대한 코딩을 분리하며 독립적으로 개발이 진행이 가능 <br/>
        - 장고의 템플릿 시스템은 HTML같은 텍스트형 언어를 쉽게 다룰 수 있도록 개발되었음 
        <br/>
    - 간단하고 확장 할 수 있는 템플릿 텍스트 처리 엔진 (template and text filtering engine)

7. 캐시 시스템 
    - 성능 향상에 도움이 되는 캐싱 시스템 <br/>
    - 자주 이용되는 내용을 저장 => 재사용 => 성능을 높힘<br/>
    - 캐시용 페이지를 메모리, 데이터 베이스 내부, 파일 시스템중 아무 곳에나 저장 <br/>
    - 캐시 단위를 페이지에서부터 사이트 전체 또는 특정 뷰의 결과, 템플릿의 일부영역만 지정하여 저장 가능 <br/>

8. 소스변경사항 자동반영 <br/>
    - 실행파일 변경내역을 바로 반영 <br/>
    - 테스트용 웹이 실행중에 소스파일을 수정 하더라도 다시시작할 필요없이 자동으로 새로운 파일이 반영 <br/>

9. 이외<br/>
    - 폼을 만들고 사용자 입력을 검사하는 유효성 검사 API<br/>
    - 확장할 수 있는 인증 시스템 <br/>
    - 다국어 지원 
    - RSS 피드를 만들어주는 피드 시스템(feed framework)<br/>

**URL을 바라보는 측면**
URL <br/>
    - RPC (Remote procedure call) <br/>
            : 클라이언트가 네트워크상에서 원격에 있는 서버가 제공하는 API 함수를 호출하는 방식 <br/>

                => URL을 전송 = 웹서버 API함수를 호출 => 인식 <br/>
    ㄴ REST (Representational State Transfer)   <br/>  
            : 웹서버에 존재하는 요소들을 모두 리소스(시간이 지나면 상태변함 <br/>
                => 조작은 HTTP메소드로)라고 정의 <br/>
                =>  URL을 통해 웹서버의 특정리소스를 표현한다는 개념 <br/>
    - 우아한 URL <br/>
        - 정규표현식을 추가적으로 사용가능함 => URL을 좀더 구체적으로 표현 가능 <br/>


# 웹 애플리케이션 서버
- 요청을 받아 처리하는 서버를 통칭 => '웹 서버' <br/>
- 세분화 <br/>
    - 웹서버(정적) <br/>
        : 웹클라이언트의 요청을 받아서 요청을 처리하고 그결과를 클라이언트에게 응답(HTML, IMG, CSS...)  <br/>
    - 웹 애플리케이션(동적)  <br/>
        : 웹서버로부터 동적페이지 요청을 받아서 요청을 처리하고, 그결과를 웹서버로 반환(프로그램실행, DB)  <br/>  
        
- **정적, 동적** 
=> 사용자가 페이지를 요청하는 시점에 페이지의 내용이 유지되는가 변경되는가를 구분하는 용어 <br/>  
    - 정적인 페이지란 ? <br/>
            - 누가, 언제 요구하더라도 항상 같은 내용을 표시하는 웹페이지<br/>
                => 주로 자바스크립트, HTML, CSS 등등... <br/>
    - 동적인 페이지란 ?  <br/>
            - 동일한 리소스의 요청이라도 누가, 언제 어떻게 요구했는지에 따라 각각 다른 내용이 반환되는 페이지를 말함  <br/>
                => 온라인 쇼핑 사이트에서 사용자마다 다른카트 내용을 보여주는 페이지  <br/>
